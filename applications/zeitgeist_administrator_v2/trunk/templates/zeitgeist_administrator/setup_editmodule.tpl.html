Editing modules

The name of the module acts as the identifier for the controller. It tries to call a class with the same name as the module. Therefor you should not choose weird module names - remember you have to name your classes accordingly.

The description of a module just acts as a reminder for yourself and is not used by the framework.

Inactive modules are ignored by the controller. If a user tries to use an inactive module, an error is thrown.
